# mk-page-pages

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 页面的集合存放与维护空间。

## 项目简介

该项目旨在集中管理和展示一组独立的 HTML 页面。每个页面可单独访问，彼此之间无强关联。仓库本身不提供任何特定主题或功能承诺，仅作为页面文件的托管与版本控制载体。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 页面归档目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── img/
├── README.md           # 本文件
└── LICENSE             # 许可证文件（如适用）
```

- **pages/** 存放所有独立 HTML 页面，文件名建议使用有意义的标识。
- **assets/** 存放页面共享的样式、脚本和媒体资源，便于统一维护。
- **index.html** 可选，可作为仓库导航或说明页。

## 页面归档说明

- 所有页面均以 `.html` 格式存放在 `pages/` 目录下。
- 每个页面应为结构完整、自包含的 HTML 文档，可独立打开并正常显示。
- 页面之间不应存在硬编码的相互依赖关系，如需共享资源，请统一引用 `assets/` 中的文件。
- 页面内容不承诺时效性或准确性，仅作为归档用途。

## 维护说明

- 如需添加新页面，请在 `pages/` 目录下创建新的 `.html` 文件，并确保其符合上述规范。
- 建议为每个页面添加简单的元信息（如 `<title>` 和 `<meta>` 标签），以便识别。
- 资源文件请放入对应子目录，避免污染根目录。
- 本仓库的维护者保留对页面内容的修改、删除或迁移权利，恕不另行通知。
